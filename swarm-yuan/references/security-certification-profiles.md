# 安全认证合规 Profile（等保4级 / BCP5级 / 0142 / PCI-DSS / ISO 27001）

> 本文件把 5 个安全认证标准的要求映射到 swarm-yuan 门禁 + 机器可检查项 + 人工核对项。
> 口径权威源：`assets/facts.conf`（FACT_CERT_PROFILES=5）。
> 门禁承载：`precheck.sh --cert-audit`（按 CERT_PROFILE 检查可机器化项；人工核对项 warn 提示）。
> 边界：认证本身需机构测评/审计，swarm-yuan 提供"门禁级可查 + 文档级锚点"，不替代机构认证。

---

## 1. 等保4级（GB/T 22239-2019 第四级安全保护要求）

> 等保2.0 第四级适用于涉及国家安全/国计民生/社会秩序的核心系统。条款号 `9.1.4.x`（第四级技术要求）+ `9.2.4.x`（管理要求）。
> 现有覆盖：§D.1 覆盖二级(7.1.4.x)/三级(8.1.4.x)，**第四级(9.1.4.x)未覆盖**。

### 1.1 第四级 vs 第三级增量要求

| 条款 | 第四级要求 | 第三级已有 | swarm-yuan 门禁 | 机器可查 |
|------|----------|----------|----------------|---------|
| 9.1.4.1 身份鉴别 | **三种以上**组合鉴别（三级=两种），全部基于密码技术 | 8.1.4.1 d 双因素 | `--authz`（缺鉴权注解） | ❌ 人工核对 MFA 实现（三种组合） |
| 9.1.4.2 访问控制 | 强制访问控制（MAC）+ 安全策略集中管控 | 8.1.4.2 自主访问控制 | `--authz`（IDOR/授权绕过） | ❌ MAC 是 OS/DB 层，门禁无法静态核查 |
| 9.1.4.3 安全审计 | 审计日志**异地实时备份** + 6个月留存 | 8.1.4.3 6个月留存 | `--shift-left` §21（可观测性日志） | 🟡 日志异地备份配置人工核查 |
| 9.1.4.4 入侵防范 | **主动入侵检测**（HIDS/NIDS）+ 恶意代码防范 | 8.1.4.4 入侵防范 | `--security`（注入/后门扫描） | ❌ IDS 是运行态设施，门禁无法静态核查 |
| 9.1.4.5 恶意代码防范 | **多层防恶意代码**（网络层+主机层+应用层） | 8.1.4.5 | `--security`/`--sbom`（供应链） | ❌ 运行态设施 |
| 9.1.4.6 可信验证 | **可信根**（基于可信芯片 TPM/TCM 的启动/运行可信验证） | 三级无 | — | ❌ 硬件级，门禁无法核查 |
| 9.1.4.7 数据完整性 | **密码技术保证**传输+存储完整性 + 系统管理员操作审计 | 8.1.4.7 | `--crypto`（SM3 国密哈希） | 🟡 密码技术使用核查（--crypto gm） |
| 9.1.4.8 数据保密性 | **国密 SM4** 加密传输+存储 | 8.1.4.8 | `--crypto`（SM4 正向核查） | ✅ `--crypto CRYPTO_PROFILE=gm` |
| 9.1.4.9 数据备份恢复 | **异地灾备** + 灾难恢复**不低于4级**（参见BCP5级） | 8.1.4.9 数据备份 | `--shift-left` §20（回滚预案） | 🟡 灾备配置人工核查（见 §2 BCP5级） |
| 9.1.4.10 剩余信息保护 | **密码技术擦除**存储介质剩余信息 | 8.1.4.10 | — | ❌ 物理介质层 |
| 9.1.4.11 个人信息保护 | **密码技术**保护个人信息 + 匿名化 | 8.1.4.11 | `--privacy`（PII 扫描） | ✅ `--privacy` + `--crypto gm` |

### 1.2 第四级机器可查项汇总（`--cert-audit CERT_PROFILE=dengbao4`）

| 检查项 | 门禁 | 姿态 |
|--------|------|------|
| 数据保密性用国密 SM4 | `--crypto CRYPTO_PROFILE=gm` | fail-closed |
| 个人信息用密码技术保护 | `--privacy` + `--crypto gm` | fail-closed |
| 数据完整性用国密 SM3 | `--crypto`（SM3 正向核查） | warn |
| 审计日志留存 ≥6个月声明 | spec §21 可观测性段 | warn |
| 灾备恢复 ≥4级声明 | spec §20 回滚+灾备声明 | warn |
| 其他（MAC/IDS/TPM/介质擦除） | — | warn（人工核对） |

## 2. 业务连续性5级（GB/T 20988-2007 灾难恢复能力等级第5级）

> GB/T 20988-2007《信息安全技术 信息系统灾难恢复规范》定义 1-6 级灾难恢复能力。第5级=实时数据传输+完整应用级切换。
> 金融 JR/T 0044-2008 §7.2.2 一类系统最低灾难恢复能力等级5级。
> 现有覆盖：domain-knowledge.md 容灾段仅速查表，**无门禁承载**。

### 2.1 BCP5级要求

| 要求 | GB/T 20988 条款 | 机器可查 | 门禁 |
|------|----------------|---------|------|
| RTO ≤6小时（一类系统） | 6.3.2 一类 | ❌ 人工核对灾备预案 | `--shift-left` §20 回滚预案声明 |
| RPO ≤15分钟（一类系统） | 6.3.2 一类 | ❌ 人工核对数据同步配置 | `--shift-left` §20 |
| 实时数据传输（第5级标志） | 附录A 第5级 | ❌ 运行态设施 | — |
| 应用级自动切换 | 附录A 第5级 | ❌ 运行态设施 | — |
| 灾备演练 ≥1次/年 | 第10章 | 🟡 spec §23 运营段声明演练记录 | `--operate` |
| 备份数据一致性验证 | 9.2.2 | 🟡 spec §23 声明 | `--operate` |

### 2.2 BCP5级机器可查项（`--cert-audit CERT_PROFILE=bcp5`）

| 检查项 | 门禁 | 姿态 |
|--------|------|------|
| spec §20 含灾备 RTO/RPO 声明 | `--shift-left` | warn |
| spec §23 含灾备演练记录声明 | `--operate` | warn |
| 灾备恢复等级 ≥4级声明（等保4级联动） | `--shift-left` §20 | warn |
| 实时数据传输/自动切换（运行态） | — | warn（人工核对） |

## 3. ISO 0142 认证

> **说明**：仓库中无明确名为"0142"的认证标准。经核查，最可能的对应是 **GB/T 22240-2020《信息安全技术 网络安全等级保护定级指南》**（与等保4级配套，决定系统安全保护等级）。若指其他标准（如 ISO/IEC 27001 的某子集编号），请在配置时修正。
> 若指 GB/T 22240-2020，其要求是"系统定级 → 备案 → 按级保护"，门禁可查的是**定级文档存在性 + 保护等级声明**。

### 3.1 机器可查项（`--cert-audit CERT_PROFILE=gb22240`）

| 检查项 | 门禁 | 姿态 |
|--------|------|------|
| spec §22 标准合规段含等保定级声明 | `--compliance` | warn |
| 定级文档存在（docs/定级报告.md 或 SPEC_FILE 含"安全保护等级"） | `--compliance` | warn |
| 定级等级与 CERT_PROFILE 一致 | `--compliance` | warn |

## 6. --cert-audit 门禁（`precheck.sh --cert-audit`）

按 `CERT_PROFILE` 环境变量/comf 配置选择认证标准，执行对应机器可查项。

### 6.1 配置

```bash
# precheck.compliance.conf 或环境变量
CERT_PROFILE=dengbao4    # dengbao4 | bcp5 | gb22240 | pcidss | iso27001 | all
```

### 6.2 行为

- 逐项检查机器可查项（上表标 ✅/🟡 的）
- fail-closed 项未通过 → fail
- warn 项未通过 → warn
- 人工核对项（标 ❌）→ 列出提示清单
- CERT_PROFILE=all → 全部 5 标准可机器化项都检查
- CERT_PROFILE 未配置 → skip（不阻塞，与现有 compliance 门禁姿态一致）

### 6.3 边界

- 认证本身需机构测评/审计（等保测评机构、PCI QSA、ISO 27001 认证机构）
- swarm-yuan 提供的是"门禁级可查 + 文档级锚点"，不替代机构认证
- 运行态设施（IDS/MAC/TPM/灾备实时切换/网络隔离）为人工核对项，门禁只 warn 提示
