# 认知偏差防范与思维模型图谱 (Cognitive Bias & Mental Models)

本文件是 swarm-yuan 第四层"认知偏差防范"的**参考图谱**，供生成的目标技能在 spec §16 与 workflow 偏差检查锚点中引用。核心理念：**认知偏差不是随机犯错，而是大脑在特定情境下的系统性捷径——可预测、可防范。**

> 本文件为方法论 reference，不依赖任何具体项目内容。

## 一、认知偏差五维分类与工程映射

### 1. 感知偏差（注意力/记忆过滤导致信息失真）

| 偏差 | 机制 | 工程触发 | 防范锚点 |
|------|------|---------|---------|
| 确认偏误 | 只看支持自己假设的证据 | 探查时只找支持"复用 X"的证据 | 三路并行子代理（强制找反例）+ `--cognition` ①概念 |
| 锚定效应 | 被第一个信息锚定 | 被第一个工期估计锚定 | spec §1.2 目标须 SMART + 多源估计 |
| 可得性启发 | 高估易回忆的事件 | 高估最近见过的 bug 类型 | 代码图谱系统性盘点（非随机 grep） |
| 选择性注意 | 只看符合预期的部分 | 评审时只看功能不看安全 | 5 维度审查 + 逻辑剃刀 |

### 2. 记忆偏差（回忆被重构）

| 偏差 | 机制 | 工程触发 | 防范锚点 |
|------|------|---------|---------|
| 后见之明偏误 | "早就知道会这样" | 复盘时觉得 bug "显而易见" | state-machine 记录原始决策 + ADR 记录时点判断 |
| 玫瑰色回忆 | 美化过去 | "老代码比现在好" | `--cognition` 六维动力学基线（用数据对比） |
| 错误归因 | 记错原因 | 把成功归自己、失败归环境 | ADR 记录决策理由 + 复盘对照 |

### 3. 社会偏差（群体压力/权威）

| 偏差 | 机制 | 工程触发 | 防范锚点 |
|------|------|---------|---------|
| 群体思维 | 为和谐不反对 | 评审会上不敢反对资深者 | subagent 上下文隔离（无群体压力）+ 逻辑剃刀"不得全盘肯定" |
| 权威偏误 | 盲信权威 | "架构师说的就对" | 逻辑剃刀"观点镜像"（只看论证不看人） |
| 从众效应 | 跟风 | "大家都用这个框架" | ADR 须记选型理由 + 替代方案 |
| 旁观者效应 | 别人会管 | "测试/安全有人管" | `--test`/`--security` 硬门禁 |

### 4. 决策偏差（损失厌恶/框架效应）

| 偏差 | 机制 | 工程触发 | 防范锚点 |
|------|------|---------|---------|
| 沉没成本谬误 | 不愿弃已投入 | 不愿弃用坏代码 | `--stable-diff` 阻断擅改 + spec §2 决策须记"是否考虑推倒重来" |
| 损失厌恶 | 宁可不损失 | 不愿重构（怕引入 bug） | spec §14 交付衰减分析（量化不重构的衰减） |
| 框架效应 | 表述影响决策 | "90% 成功" vs "10% 失败" | spec §2 决策记录须含正反两面 |
| 现状偏见 | 倾向现状 | "能跑就别动" | `--reuse` 复用检索 + `--layer` 结构检查 |
| 过度自信 | 高估自己 | "我能改这个核心模块" | `--cognition` 认知总分（低分=高发） |
| 规划谬误 | 乐观估计 | 工期永远低估 | spec §4 tasks 须含缓冲 + §14 衰减分析 |

### 5. 元认知偏差（对自身认知的认知失真）

| 偏差 | 机制 | 工程触发 | 防范锚点 |
|------|------|---------|---------|
| 达克效应 | 不懂的人过度自信 | 新手自信改核心 | `--cognition` 认知总分 + spec §16.3 "承认不确定性" |
| 能力错觉 | 把运气当能力 | "上次没出事故这次也不会" | `--impact` 消费方反查 + 回归测试 |
| 双重标准 | 对人严对己松 | 审查别人代码严、自己松 | 逻辑剃刀"灵魂拷问" + subagent 两阶段审查 |
| 盲点偏见 | 看到别人的偏差看不到自己的 | "我没有确认偏误" | 四层认知基底体检（`--cognition` 四层总分） |

## 二、工程阶段偏差检查锚点

目标技能 workflow 须在以下节点插入偏差自检：

| workflow 节点 | 高发偏差 | 自检问题 |
|--------------|---------|---------|
| ①需求理解 | 锚定/确认偏误 | 我是否被第一个方案锚定？搜过反例吗？ |
| ②设计 spec | 现状偏见/沉没成本/规划谬误 | 现有代码值得保留吗？工期乐观吗？考虑过推倒重来吗？ |
| ③实施 plan | 过度自信/可得性 | 任务拆分基于最近记忆？低频高风险子任务遗漏？ |
| ⑤编码实现 | 达克/双重标准 | 我对这个库的理解足以自信修改？审查标准同样用于自己？ |
| ⑥测试审查 | 后见之明/群体思维 | 测试通过是"预期中"还是"真验证"？因作者资历放松标准？ |
| ⑧构建发布 | 损失厌恶/过度自信 | 回滚方案真可行？对"不会出事"的信心有证据？ |

## 三、思维模型 8 类检查清单（spec §16.2 须对照 3+ 条）

### 1. 第一性原理思考
- **核心**：剥到不可再剥的物理约束，从公理重建
- **工程检查**：这个需求的最底层物理约束是什么？哪些假设是"一直这样做"而非"必须这样做"？
- **反偏差**：防锚定效应、现状偏见

### 2. 系统思维
- **核心**：反馈回路、瓶颈约束、涌现、杠杆点
- **工程检查**：这个改动的二阶效应？瓶颈在哪？哪个点撬动全局？
- **反偏差**：防合成谬误、单因谬误

### 3. 概率思维
- **核心**：贝叶斯更新、基础概率、期望值、幂律
- **工程检查**：成功先验概率？新证据如何更新？在赌幂律尾部吗？
- **反偏差**：防过度自信、可得性启发

### 4. 经济学思维
- **核心**：机会成本、边际效益、比较优势、规模经济
- **工程检查**：选 A 放弃了什么？多投入一单位边际收益？谁有比较优势？
- **反偏差**：防沉没成本、损失厌恶

### 5. 反脆弱思维
- **核心**：从混乱中受益、杠铃策略、冗余、非对称风险
- **工程检查**：架构能从故障中受益？最坏情况非对称？有冗余兜底？
- **反偏差**：防涅槃谬误、假两难

### 6. 心智模型多元
- **核心**：多元思维格栅、可证伪性、奥卡姆剃刀
- **工程检查**：结论可证伪？最简解释？跨学科有反例？
- **反偏差**：防确认偏误、单一模型过度自信

### 7. 人性与博弈
- **核心**：囚徒困境、纳什均衡、委托代理、信任
- **工程检查**：各方激励对齐？谁是委托人代理人？零和博弈？
- **反偏差**：防群体思维、权威偏误

### 8. 认知与时间
- **核心**：复利、临界点、熵增、路径依赖
- **工程检查**：决策有复利效应？在临界点前？熵增超过治理？
- **反偏差**：防规划谬误、现状偏见

## 五、与前几层的协同

| 层 | 偏差防范角色 |
|----|-------------|
| 第一层 认知递进 | 建立完整认知，防盲点偏见（不认识就不觉风险） |
| 第二层 思维语言 | 多元思维格栅，防单一模型过度自信 |
| 第三层 认知辩证 | 逻辑剃刀对抗审查，防确认偏误+群体思维 |
| **第四层 偏差防范** | **五维偏差扫描+思维模型检查清单，防理性假设失效** |

> 四层叠加：看见→想对→证伪→纠偏。偏差防范是最后一道防线——即使前三层都做对了，系统性非理性仍可能导致错误决策。
