## 0. 适用范围与术语纠偏

**适用对象**：银行、证券期货、保险、支付清算等持牌金融机构的信息系统研发交付，以及为其提供产品/服务的信息技术系统服务机构（外包/供应商）的研发交付。

**术语纠偏（重要）**：不存在名为《个人金融信息保护法》的立法。金融领域个人信息保护的真实依据链为：

1. 《个人信息保护法》（2021，法律层）；
2. JR/T 0171-2020《个人金融信息保护技术规范》（人民银行行业标准，C3/C2/C1 分级）；
3. 《中国人民银行业务领域数据安全管理办法》（人民银行令〔2025〕第 3 号）；
4. 《银行保险机构数据安全管理办法》（金规〔2024〕24 号，金融监管总局）。

（纠偏写法沿用 R8 对「GB/T 30976」的处理先例：任务下位词与真实标准/立法不符时显式更正。）

## 4. 配置包用法与姿态

`assets/industry-profiles/finance.conf` 为 conf 片段：`cat assets/industry-profiles/finance.conf >> precheck.conf`（bash 后赋值覆盖先赋值），再按项目裁剪——占位值 `<...>` 必须替换为真实路径。姿态约定：

- 未配置静默跳过；启用后 fail-closed（`--sbom`/`--privacy`/`--crypto`/`--authz`/`--docs-pack`/`--release-sign` 均属此列）；
- 豁免必须按 standards-compliance.md §F.2 五字段格式登记（对象|规则|理由|审批人|日期），对应 GB/T 39786「宜」级条款裁剪论证要求（R8 §14.3）；
- `PRIVACY_SCAN_DIRS` 与 `DOCS_PACK_REQUIRED` 为本 profile 必填裁剪项；`PRIVACY_SCAN_DIRS` 启用但配置目录全不存在 → fail（fail-closed 语义，切勿带占位值上生产）；
- 本机无 cosign/gitleaks/semgrep 时工具路径用 fixture 内 mock 验证；`SENSITIVE_TOOL`/`SECURITY_TOOL` 保持 `auto` 自动降级内置正则。
