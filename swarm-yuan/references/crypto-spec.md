# 密码学应用规范（对齐 GB/T 39786 / ISO/IEC 18033 / NIST SP 800）

> 系统性密码学规范——密钥生命周期 / 算法选型（含国密 SM2/SM3/SM4）/ 随机数质量 / PKI / 后量子迁移。
> 与 `security-spec.md`（OWASP 应用安全层）互补：security-spec 管"应用怎么用密码学"，本规范管"密码学本身怎么选、怎么管"。
> 被 `check_crypto` 门禁（GB/T 39786 密评弱算法扫描 + 国密正向核查）作为判定依据。**机构密评测评属线下**——本规范与门禁输出不构成密评合规证据。
>
> 调研依据：`docs/research/R8-security-standards.md`、verifier 标准合规探索报告 §5.3（密码学规范分散、无独立文档确认）。

---

## 1. 总原则

- **最小化**：只加密必要数据；能用哈希就不用可逆加密，能不落地就不落地。
- **默认安全**：安全配置为默认值（如 TLS 默认开、安全随机源默认用）；弱化须显式豁免留痕。
- **密钥与代码分离**：禁止硬编码密钥/口令/证书私钥（与 `--sensitive` 门禁协同）；密钥进 KMS/HSM/密钥管理服务。
- **不自造算法/协议**：只用公开审定算法与成熟库实现；禁止自研加密算法、自实现 TLS。

## 2. 密钥生命周期

| 阶段 | 要求 |
|---|---|
| 生成 | CSPRNG 生成（见 §4）；密钥长度符合 §3 选型；禁止由口令直接截取当密钥（须 KDF：PBKDF2/scrypt/argon2/HKDF） |
| 分发 | 安全信道（TLS/带外）；禁止明文密钥随代码/镜像/制品分发 |
| 存储 | KMS/HSM/操作系统密钥链；应用内只存引用（密钥 ID）；禁止入库、入仓、入日志 |
| 轮换 | 定期轮换 + 泄露即时轮换；轮换窗口内新旧并存期须受控 |
| 撤销 | 泄露/离职/下线即撤销；证书撤销走 CRL/OCSP（见 §5） |
| 销毁 | 安全擦除（内存清零、介质消磁/加密擦除）；备份介质同步处置 |

## 3. 算法选型

| 用途 | 推荐 | 禁用 |
|---|---|---|
| 口令哈希 | bcrypt / scrypt / argon2（带盐、足量代价因子） | MD5 / SHA1 / 无盐哈希 |
| 对称加密 | AES-256-GCM（AEAD） | DES / 3DES / RC4 / ECB 模式 |
| 非对称 | RSA-3072+ / ECDSA P-256 / Ed25519 | RSA-1024 |
| 哈希/完整性 | SHA-256 / SHA-3 族 | MD5 / SHA1 |
| **国密（GB/T 39786-2021 密评场景）** | **SM2（非对称/签名/密钥交换）/ SM3（哈希）/ SM4（对称，GCM/CTR 等安全模式）** | 密评场景用 AES/RSA/SHA-2 替代国密 |

**国密使用模式要点**（密评场景）：
- 签名/验签：SM2 签名（GM/T 0009）；密钥交换：SM2 密钥交换协议。
- 数据加密：SM4 + 安全模式（禁止 ECB）；完整性：SM3 或 HMAC-SM3。
- SSL/TLS：国密双证体系（TLCP，GM/T 0024），签名证书 + 加密证书分离。
- 随机数：密评场景随机数发生器须符合 GM/T 0005。

## 4. 随机数质量

- 安全场景（密钥/token/盐/nonce/会话 ID）**必须 CSPRNG**：Linux `/dev/urandom`、Python `secrets`、Java `SecureRandom`、Node `crypto.randomBytes`、Go `crypto/rand`。
- **禁用**于安全上下文：`Math.random`（JS）、`rand()`/`random`（Python/C 伪随机）、`java.util.Random`、时间戳/自增 ID 充当随机源。
- 门禁协同：`check_crypto`（gm profile）对安全上下文的弱随机数用法做 warn 级提示。

## 5. PKI 与证书管理

- 证书链**完整验证**（禁止跳过链校验、禁止 trust-all 实现）。
- 撤销检查：OCSP（推荐装订 OCSP stapling）或 CRL。
- 有效期监控：到期前告警 + 自动化续期；私钥保护（HSM/密钥链，禁止明文落盘）。
- 国密 PKI：双证书体系（签名证书 + 加密证书），TLCP 握手。

## 6. 后量子迁移（PQC）展望

- NIST 已定标：**FIPS 203（ML-KEM，密钥封装）/ FIPS 204（ML-DSA，签名）/ FIPS 205（SLH-DSA，哈希签名）**。
- 迁移策略：先盘点密码资产（crypto-agility），过渡期用**混合模式**（经典算法 + PQC 并行，如 X25519+ML-KEM）。
- 长期保管数据（"先存后解"风险）优先迁移。

## 7. 对齐标准

| 标准 | 本规范对应 |
|---|---|
| GB/T 39786-2021（信息系统密码应用基本要求，密评） | §3 国密选型 + §2 密钥生命周期 + §4 随机数；门禁落点 `check_crypto` |
| ISO/IEC 18033（加密算法族） | §3 通用算法选型依据 |
| NIST SP 800-131A（算法迁移与密钥长度） | §3 禁用清单（RSA-1024/SHA1/DES 退役依据） |
| NIST FIPS 203/204/205（PQC） | §6 后量子迁移 |
| OWASP A02（Cryptographic Failures） | §1 总原则与 `security-spec.md` §1.6 衔接 |
