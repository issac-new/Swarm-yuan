# 规格说明（compliant 样本）

## 1. 背景

订单子系统重构规格。

## §19 测试左移设计

### 19.1 测试策略

用例先于实现编写，按正常/边界/异常三路径覆盖。

### 19.2 用例骨架

| 用例 | 输入 | 预期 | 类型 |
| --- | --- | --- | --- |
| 正常下单 | 合法订单 | 下单成功 | 正常 |
| 库存边界 | 库存恰好为 1 | 下单成功 | 边界 |
| 库存不足 | 库存为 0 | 拒绝下单 | 异常 |

### 19.3 覆盖率目标

行覆盖率不低于 80%。

## §21 可观测性约束

关键链路须埋点：下单计数指标、订单状态变更日志、跨服务调用链标识。

## 回滚预案

- 回滚方式：发布失败时切回上一版本镜像。
- 验证：回滚后核心下单链路冒烟用例全绿。
- 窗口：发布后 30 分钟内可一键回退。
